Terminal Setup · 照抄即用清单

Ghostty 终端配置清单

这是我这台 Mac 上真实在跑的全套终端配置，按「七层」分好类。每一项都有一句话说明 + 一行可复制的安装命令。目标只有一个：把终端从一个吓人的黑窗口，变成又快又顺手的工作台。Windows 同学请走 Windows Terminal + WSL2，本清单针对 macOS。想先弄懂每一层「为什么这么配」→ 七层归档总览。

发布于 2026-06-12
1
先装 Homebrew
macOS 的「命令行软件管家」，是总开关。照下面 第 0 步 装好它，后面的 brew install 才跑得通。
2
交给 Claude Code
把某一项的命令复制，对 Claude Code 说：帮我装这个并配好 .zshrc，它会逐项搞定。
3
改完重载
动过 .zshrc 后执行 source ~/.zshrc 或新开一个窗口生效。
第 0 步 · 必装Homebrew —— 所有工具的「总开关」，没它后面全装不了
# 装 Homebrew（中途会让你按一下回车、输一次开机密码）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
# 装完让这个终端立刻认识 brew（Apple Silicon / M 系列芯片）
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
装好后用 brew --version 验证。Intel 芯片的老 Mac 把上面两处 /opt/homebrew 改成 /usr/local。
⚠️ 例外：Oh My Zsh（curl 装）、两个 zsh-* 插件（git clone）、claude-hud（Claude Code 里装）不走 Homebrew，其余全靠它。
第 1 步 · 全家桶一行把现代 CLI 全家桶全装齐（先完成第 0 步）
# 终端 + prompt + 导航 + 全家桶，一次装完
brew install starship eza bat fd ripgrep fzf zoxide yazi autojump
brew install --cask ghostty font-jetbrains-mono-nerd-font
这一步只装「能用 brew 装的」。Oh My Zsh 和两个 zsh 插件请到下面对应分层里单独复制安装命令。
01

地基层 · 壳与框架

终端窗口 + shell 插件管理

这两个是地基：Ghostty 是那个会发光的窗口（GPU 渲染、分屏、字体、主题都归它管）；窗口里跑的是 zsh，Oh My Zsh 是 zsh 的「插件 + 别名管理框架」，后面的插件都挂在它上面。

Ghostty终端窗口又快又干净的现代终端，GPU 加速、原生分屏、配置就是一个文本文件
brew install --cask ghostty
配置文件在 ~/.config/ghostty/config，存盘即时生效，无需重启。字体/主题/快捷键见下面对应分层。
Oh My Zsh插件框架管理 zsh 的插件与别名，是 git / z / 自动补全等插件的容器
sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"
装完在 ~/.zshrc 里把要用的插件填进去：plugins=(git z node npm zsh-autosuggestions zsh-syntax-highlighting sudo history)。⚠️ 主题留空 ZSH_THEME=""，prompt 交给 starship。
02

颜值层 · 提示符 / 字体 / 主题

让终端好看又好读

这三样决定「第一眼观感」。starship 接管那行带目录和 git 分支的提示符；Nerd Font 是所有图标能显示的前提（没它 eza 图标、git 分支符号全是方块）；主题负责配色。

💡
本课提示符统一用 starship：跨 shell 通用、配置就一份 starship.toml，比 p10k 等老方案更省心。那行带目录和 git 分支的提示符，就是它画的。
starship提示符跨 shell 的提示符引擎，自动显示目录 / git 分支 / 语言版本 / 时间
brew install starship
在 ~/.zshrc 末尾加 eval "$(starship init zsh)"；样式配置在 ~/.config/starship.toml（我用的是带色块的 powerline 风格，可直接照抄 ↓）。
▶ 展开我的完整 ~/.config/starship.toml · 直接整段照抄
⚠️ 那些方块色块图标需要 Nerd Font 才显示；请先装好上面的字体。

JetBrainsMono Nerd Font字体带图标字形的等宽字体，图标 / 分支符号 / 文件图标全靠它
brew install --cask font-jetbrains-mono-nerd-font
在 Ghostty config 里指定：font-family = "JetBrainsMono Nerd Font"、font-size = 18。这是颜值层的地基，建议第一个装。
主题 / 配色外观浅色 GitHub Light + 深色 Catppuccin Mocha，跟随系统自动切换
# 写进 ~/.config/ghostty/config
theme = light:GitHub Light Default,dark:Catppuccin Mocha
主题是 Ghostty 内置的，无需额外安装。配 minimum-contrast = 4.5 可避免浅色背景下 Claude Code 的灰字糊到看不清。
03

导航层 · 目录跳转 / 文件管理

少敲路径，可视化找文件

终端最累的就是「切目录、找文件」。zoxide / z 记住你去过的目录，敲一半关键词就跳过去；yazi（读音「鸭子」）是终端里的可视化文件管理器，像 Finder 一样用方向键浏览。

zoxide目录跳转记住历史目录，敲一半就跳。比 cd 聪明的智能跳转
brew install zoxide
在 ~/.zshrc 加 eval "$(zoxide init zsh --cmd zi)"，用 zi 关键词 跳转。我同时保留了 Oh My Zsh 的 z 插件和 autojump（brew install autojump），三套都能跳，挑顺手的用即可。
yazi文件管理终端版「Finder」：方向键浏览、图片预览、按 z 调 zoxide 跳目录
brew install yazi
关键：在 ~/.zshrc 加一个 y() 包装函数，退出 yazi 时自动把 shell 切到你最后所在的目录。yazi 内部还会调用下面全家桶里的 fd / ripgrep / fzf 做搜索，所以建议一起装。
04

现代 CLI 全家桶 · 替换老命令

2026 公认的 Rust 系利器

把上古的 ls / cat / find / grep 换成带颜色、带图标、更快的现代版。它们也是 yazi 搜索的底层依赖，一起装最省心。

eza替代 ls列目录带图标 + 颜色 + git 状态 + 树形视图
brew install eza
在 ~/.zshrc 加别名：alias ls='eza --icons --group-directories-first'、alias ll='eza -lA --icons --git --time-style=relative'、alias lt='eza --tree --level=2 --icons'。
bat替代 cat看文件带语法高亮 + 行号 + git diff 标记
brew install bat
直接 bat 文件名 即可。可选别名 alias cat='bat --paging=never'。
fd替代 find找文件，语法人话、默认跳过 .git/node_modules、还很快
brew install fd
用法 fd 关键词，比 find . -name 好记太多。也是 yazi / fzf 的搜索后端。
ripgrep（rg）替代 grep在文件内容里搜文字，极快，自动忽略二进制和 .gitignore
brew install ripgrep
命令是 rg 关键词。Claude Code 和 yazi 内部都靠它做全文搜索。
fzf模糊搜索万能模糊查找器：Ctrl+R 翻历史命令、Ctrl+T 选文件
brew install fzf
在 ~/.zshrc 加 [ -f ~/.fzf.zsh ] && source ~/.fzf.zsh 启用快捷键。配合 fd/rg 一起最强。
05

效率层 · 补全 / 高亮 / 别名

挂在 Oh My Zsh 上的插件

这些是 zsh 的「打字辅助」：边打边给灰字预测、命令对错变色、git 一堆短别名。它们写进 ~/.zshrc 的 plugins=(...) 里就生效。

zsh-autosuggestions命令预测根据历史灰字预测整条命令，按 → 补全
git clone https://github.com/zsh-users/zsh-autosuggestions ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-autosuggestions
克隆完把 zsh-autosuggestions 加进 plugins=(...)。
zsh-syntax-highlighting语法高亮命令对了变绿、错了变红，敲错当场看出来
git clone https://github.com/zsh-users/zsh-syntax-highlighting.git ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-syntax-highlighting
同样加进 plugins=(...)，建议放在列表最后。
git / history / sudo …内置插件Oh My Zsh 自带，无需安装，写进 plugins 即用
# 直接编辑 ~/.zshrc 的 plugins 行
plugins=(git z node npm zsh-autosuggestions zsh-syntax-highlighting sudo history)
git 给一堆短别名（gst=status、gco=checkout）；history 查历史命令；sudo 连按两下 Esc 给上条命令加 sudo。
06

监控层 · Claude Code 状态栏

不是终端插件，是 Claude Code 的

claude-hud 装在 Claude Code 里、不是 zsh 插件，所以单独归到「监控层」。作用是在 Claude Code 底部状态栏显示模型、上下文占用、5 小时 / 7 天额度恢复时间、当前任务等。

claude-hudCC 状态栏监控模型 / 上下文占比 / 额度恢复时间 / 当前任务
# 在 Claude Code 里输入斜杠命令
/plugin
在 Claude Code 的 /plugin 插件市场里找到 claude-hud 安装，再把它设为 statusLine（我的 ~/.claude/settings.json 里已配好命令）。它跟终端无关，属于 AI 工具层。
07

窗口管理 · Ghostty 快捷键

无需安装，写进 config 即用

Ghostty 自带分屏 / 标签 / 下拉终端，不需要 tmux。下面这组快捷键写进 ~/.config/ghostty/config 就能用。

分屏 / 标签 / 字号keybind一屏拆多块、多开标签、随手缩放
# 节选，完整版见我的 config
keybind = cmd+u=new_split:right
keybind = cmd+shift+d=new_split:down
keybind = cmd+t=new_tab
keybind = cmd+shift+f=toggle_split_zoom
cmd+u 右分屏、cmd+shift+d 下分屏、cmd+shift+f 把当前分屏临时放大到全窗口（再按还原）。
Quick Terminal（下拉终端）全局热键任意 App 里按 cmd+` 从屏幕顶部滑下一块临时终端
keybind = global:cmd+backquote=toggle_quick_terminal
quick-terminal-position = top
用 global: 前缀，所以不在 Ghostty 时也能呼出。随手敲个命令、关掉不打断手头的活。
