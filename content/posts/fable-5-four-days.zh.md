# 我把 Fable 5 用到提前下线

前几天打开 Claude Code，顶上弹出来一行字。

<figure>
  <img src="assets/posts/fable-5-four-days/card-banner.png" alt="Fable 5 上线横幅：算在套餐里，用到 6 月 22 号">
  <figcaption>它自己弹的横幅：「算在套餐里，用到 6 月 22 号，之后走 API。」</figcaption>
</figure>

意思是：来了个新模型，主打复杂的、要跑很久的活；22 号之前，用它不额外花钱，算在我套餐额度里；22 号之后想接着用，就走 API 按量付费。

我盯着「complex, long-running work」这几个字，有点走不动道——我手上的活全是这种。能跑很久、要拆很多步、中途不能断的长任务，正是平时最难伺候的一类。

于是我做了个决定：趁它还在套餐里，往死里用。

## 平时我根本用不完

先说个背景。我订的是最高的那档套餐。

这个额度有多宽裕？平时我一整周高强度用下来，那个「7d 周额度」的进度条，也就走到五十几个百分点。从来没碰过上限，甚至懒得看它。

所以这次我心里是有底的：就算猛用，一周的额度也够我造的。

我错了。

## 我到底喂了它什么

我不是开一个窗口慢慢聊。我是这么干的：

<figure>
  <img src="assets/posts/fable-5-four-days/card-parallel.png" alt="六个会话并排，多 agent 同时跑 Fable 5">
  <figcaption>六个会话并排开，每个里面再挂多 agent 的工作流，清一色 Fable 5。</figcaption>
</figure>

六个会话并排开，各跑各的互不打扰；其中还挂着多 agent 的工作流——一条命令下去，底下分四五个 agent 同时调研、同时干活。把最重、最长、最烧 token 的任务全塞给它，一天到晚不停。

这种用法，token 是按倍数烧的。一个普通对话烧 1 份，六开并行 + 多 agent，烧的是十几份、几十份。

## 进度条开始往上窜

然后我就看着那个平时懒得看的周额度，第一次有了存在感。

<figure>
  <img src="assets/posts/fable-5-four-days/card-climb.png" alt="7d 周额度 88% → 94% → 见底">
  <figcaption>「7d」周额度：88% → 94% → 见底，三天就走完。</figcaption>
</figure>

第三天傍晚，88%。我愣了一下——平时一周到不了 60，这才三天就 88 了。

心想「那就再用用呗，反正用得值」。几个小时后再看：

<figure>
  <img src="assets/posts/fable-5-four-days/card-cover.png" alt="顶配套餐，三天就被榨干">
  <figcaption>平时一周用不完的额度，这次三天多，被我榨得干干净净。</figcaption>
</figure>

94%。再往后，见底。平时一周用一半都嫌多的额度，这次三天多，被我榨得干干净净。

## 然后，它提前走了

按横幅上写的，22 号之前都算套餐额度，之后走 API。我盘算得挺好：周额度用光也不慌，22 号前接着白嫖，22 号后实在想用就掏 API 的钱，总归还能再陪它一阵。

结果没到 22 号，它就没了。

不是「我额度用完了」那种没——是模型本身下线了，API 也调不到。它比自己印在横幅上的那个日期，走得还早。

一个我用了三四天、刚摸到点脾气、知道该怎么跟它说话的模型，说没就没了。以后也不会再有。

## 模型不是软件，是生鲜

这件事让我改了一个一直没意识到的默认假设。

我过去默认：工具是恒定的。装上一个软件，它就一直在那儿，今天能开明天也能开，除非我自己删掉它。

但模型不是软件。模型更像生鲜——它有保质期。而且标签上印的那个「保质期」，还是个乐观估计，随时可能提前。

想通这点，我反而一点不后悔那三天的猛用。

你设想另一个版本：我悠着点，把额度省着花，告诉自己「不急，能用到 22 号呢」。那今天我手里会是什么？什么都没有。额度到周末会清零，一分没省下；模型提前下线，一秒没多用出来。两头空。

省下来的额度不属于你，下线的模型也不属于你。**真正属于你的，只有你已经花掉、已经变成产出的那部分。**

## 所以

如果你也是那种「好东西舍不得用、爱留着」的人，这事可以当个提醒：

当一个东西又好、又限量、又随时可能消失，最大的浪费不是用太猛，是犹豫。

我猛用了四天，它还是先走了。但那四天跑出来的东西，留下了。
