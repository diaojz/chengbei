# The "Basics" Were Never About Code

Last week a designer who can't write a line of code shipped an internal tool in two days. Her team actually uses it.

She did exactly three things: described the pain point clearly, said "right" or "wrong" to each iteration, and made the calls at key decision points. Never touched the code.

People assume the learning curve for AI coding is six months — syntax, debugging, Stack Overflow. It's two days, because what you need to learn isn't any of that. It's how to be a competent CEO: define the problem, delegate the execution, review the result.

The "basics" in "zero-basics" were never about code.
