# Claude 登录常见问题 · FAQ

按报错现象查。找到一样的就照着做。还不清楚就发我。

> 没读过 [Claude 账号登录完整方案](#/p/claude-login-guide) 的同学先回去把那篇看完，里面讲了为什么会出问题、有哪三条路可走。这一篇只解决具体卡点。

---

## Q1 · 注册时收不到邮箱验证码

**先排除：**

- 看一眼**垃圾邮件**文件夹（90% 在这里）
- Gmail 用户在「所有邮件」搜 `anthropic`
- 等 5 分钟再试一次发送（频繁点会被限流）

**如果还是没有：**

- 换一个邮箱重新注册（推荐 Gmail / Outlook，不要用 QQ / 163 / 国内企业邮箱）
- 注册时确保**梯子开着**，并且节点不是国内出口

---

## Q2 · 注册成功，但用一两天账号就被 suspend（账号被封）

99% 是因为 **IP 不干净**或**IP 漂移**：

- 用了机场 / 共享节点 → IP 段被几千个人用过，已经在 Anthropic 黑名单里
- 节点自动切换（比如自动选最快节点）→ 系统判定异地登录

**解决：**

- 短期：换一个独立住宅 IP 重新注册一个号
- 长期：**别折腾了，直接走中转网关**（路线 A），不需要官方账号也能用全套 Claude Code

---

## Q3 · Claude Code 命令行报 401 Unauthorized

按这个顺序排查：

1. **Key 没复制全**：从控制台复制 Key 时容易少一两位。重新复制粘贴一次。
2. **环境变量没生效**：终端里跑 `echo $ANTHROPIC_API_KEY`，能打印出 Key 才算成功。打不出来说明 `export` 没写进 shell 配置文件。
3. **BASE_URL 写错了**：必须是 `https://gw.chengbei.org/api`（注意结尾的 `/api`，少了就是 401）
4. **Key 在网关后台被禁用了**：去 `gw.chengbei.org` 控制台看下 Key 状态

---

## Q4 · Claude Code 报 403 Forbidden

通常是**模型权限问题**：

- 你的 Key 没开 Claude Sonnet / Opus 的访问权限
- 解决：进 `gw.chengbei.org` 控制台 → Keys → 编辑这个 Key → 勾上需要的模型 → 保存

---

## Q5 · 网页 claude.ai 能用，Claude Code 命令行不能用

这两个用的是不同的鉴权方式：

- **网页**：浏览器 Cookie + 你的账号 session
- **Claude Code**：API Key（要么 Anthropic 官方 Key，要么走中转）

网页能用不代表命令行能用。**用命令行就老老实实配 API Key**（推荐走中转网关）。

---

## Q6 · 用网页 claude.ai 一会儿就「Network error」/ 一直转圈

- 切节点试试，优先选**日本 / 新加坡 / 美西**的独享节点
- 浏览器开个隐身窗口重新登录（排除 Cookie 污染）
- 关掉所有浏览器扩展再试（特别是隐私类、广告拦截类）
- 还不行，说明这个 IP 已经被 Anthropic 风控，**换 IP 或者改用中转**

---

## Q7 · 我想订 Claude Pro 但付款过不去

Anthropic 不接受中国大陆发卡的银行卡。可选方案：

- **WildCard 虚拟信用卡**：国内最常见的解决方案，开卡费 ~¥15，每月手续费有
- **Depay / OneKey Card**：USDT 充值，更灵活
- **找朋友代付**：海外信用卡直接付，把 Pro 绑定在你的账号

**但是**：很多学员用 WildCard 付款 Pro 之后照样被风控、被 suspend。所以**先评估你的真实需求**——如果只是想用 Claude Code 写代码，**走中转网关比订 Pro 更稳**。

---

## Q8 · 我已经买了 Pro，怎么和中转一起用

两个一起留着，按场景分：

- **网页对话、看图、长聊** → 用官方 Pro（claude.ai）
- **终端写代码（Claude Code）** → 走中转网关（`gw.chengbei.org`）

这样官方 Pro 的额度专门留给网页对话，命令行高频写代码不会卡你的 Pro 限额。

---

## Q9 · 中转网关贵不贵？计费怎么算？

按 token 用量计费，没有月费。日常使用一个学员**月均 ¥10-50 之间**（看你写代码的频次）。

进 `gw.chengbei.org` 控制台能看到每个 Key 的实时余额和消耗记录。

---

## Q10 · 中转网关会不会哪天关掉？我的代码上下文怎么办？

- 中转网关只是个代理，**不存你的对话内容**。你的代码上下文存在 Claude Code 本地（`~/.claude/projects/`），跟网关没关系。
- 哪天我不维护了，你直接换别的中转或者换成官方 Key，Claude Code 的所有项目历史不受影响。
- 网关本身有 SLA 监控，目前线上稳定运行。
