# 模型也有保质期

前几天 Claude Code 里弹出来一行字：

> Fable 5 is here! Our newest model for complex, long-running work. Included in your plan limits until Jun 22, then switch to usage credits to continue.

翻译过来：新模型，专治复杂的、跑很久的活；6 月 22 号之前算在你套餐额度里，之后想接着用就走 API 计费。

我看到「complex, long-running work」就两眼放光——我手上全是这种活。于是把它当宝贝一样用。

## 我有多上头

我订的是最大的那档套餐。平时一整周，额度也就用到五十几个百分点，从来没紧张过。

这次不一样。我同时开了六个会话并排跑，每个里面再挂多 agent 的工作流，把最重、最长、最烧 token 的活全喂给它。一天到晚不停。

然后我盯着状态栏上那个「7d」周额度的数字，眼看着它往上窜：

- 第三天傍晚，88%。
- 几个小时后，94%。
- 再往后，见底。

平时一周用一半都用不完的额度，这次三天多就榨干了。

## 然后它提前走了

标签上写的是 6 月 22 号。我本来盘算着：套餐额度用光也没事，22 号之前还能接着用，22 号之后走 API 也行，总归还能再摸一阵。

结果没到 22 号，它就没了。

不是「额度用完了」那种没了——是模型本身下线了，API 也调不到。我攒着的那点「之后再用」的念想，扑了个空。

一个我用了三四天、刚摸到点脾气的模型，说走就走，比它自己印的那个日期还早。以后也不会再有了。

## 模型也有保质期

我过去默认工具是恒定的。装上一个软件，它就一直在那儿，今天能用明天也能用。

但模型不是软件。模型更像生鲜——它有保质期，而且标签上印的那个日期，是乐观估计。

想明白这件事，我反而一点不后悔那三天的猛用。

如果当时我悠着点、把额度省着花，想着「反正能用到 22 号」，那我现在手里什么都没有——既没省下额度（它会清零），也没多做出东西（模型没了）。

真正属于你的额度，从来只有你今天花掉的那部分。

好东西、又是限量的，唯一的浪费是犹豫。
