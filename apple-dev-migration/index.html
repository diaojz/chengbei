Apple Developer · Mac Migration Guide

换电脑：
开发者账号怎么搬

开发者账号本身在云端，跟 Apple 账户走，换 Mac 不受影响。真正要操心的只有一样：存在旧电脑钥匙串里的签名证书私钥。按 2026 年最新规则，给你三条迁移路线和分证书类型的处理办法。

01

核心结论：先安心，再动手

云端的不用搬 · 本地的只有一样
☁️ 全部在云端 · 不用搬
账号、App、收款，跟着 Apple 账户走
开发者会员资格、App Store Connect 里的 App / 审核状态 / 收款 / 订阅 / TestFlight，全部存在苹果服务器。新电脑登录同一个 Apple 账户即恢复，零迁移动作。
🔑 唯一在本地 · 要管
钥匙串里的证书私钥
签名证书的私钥只存在你旧电脑的钥匙串，苹果服务器上没有备份。丢了不致命（多数证书可重建），但提前花 5 分钟备份，能省掉后面所有麻烦。
02

先更新认知：四个新事实

老教程很多已经过时 · 2026-06 查证
新事实 01 · ⚠️
Xcode 16 移除了「Export Developer Accounts」
老教程里「Xcode 一键导出账号」的菜单已经不存在了。手动迁移证书只剩一条路：钥匙串访问导出 .p12。
新事实 02 · ☁️
Xcode 13 起，发布走「云签名」
走 Organizer 上传 App Store 时，发布证书私钥存在苹果云端（cloud-managed），本机没有私钥，换机零迁移。
新事实 03 · 📛
Apple ID 已更名「Apple 账户」
只是改名，账号体系不变。本指南里说的 Apple 账户，就是你原来的 Apple ID。
新事实 04 · 📂
描述文件缓存目录变了
Xcode 16 起本地缓存挪到 ~/Library/Developer/Xcode/UserData/Provisioning Profiles。CI 脚本写死老路径的要更新；手动换机不受影响。

03

路线 A · 迁移助理

三条路线 · 1 / 3 · 最省心
🧳
路线 A · 零操作
整机搬家，
钥匙串跟着走
macOS 自带的「迁移助理」会把钥匙串（含全部证书私钥）、Xcode 配置一起搬到新机。这是唯一一条「什么都不用懂」的路线。
适合谁
新旧两台都在手上不介意旧环境整体搬0 条命令
一句话：私钥住在钥匙串里——钥匙串搬走了，签名能力就跟着走了。
4 个要点
  • 钥匙串整体迁移：登录钥匙串连同里面的证书 + 私钥原样到达新机，无需任何导出动作
  • 新机首次打开 Xcode：到 Settings → Accounts 重新登录一次 Apple 账户（登录态不随迁移走）
  • 可能要输旧机密码：首次访问迁移过来的钥匙串项时，系统可能要求输入旧机的登录密码解锁
  • 验证方式：打开项目跑一次 Archive，签名不报错即迁移成功
03

路线 B · 手动导出 .p12

三条路线 · 2 / 3 · 最干净
🔑
路线 B · 约 5 分钟
只搬私钥，
其余全新开始
新机想要干干净净从零装环境，又不想丢证书？用「钥匙串访问」把证书连私钥导成 .p12 文件，拷到新机双击导入。
关键提醒
Xcode 16 无导出菜单钥匙串是唯一途径
顺手做对一件事：.p12 同时存一份进密码管理器（如 1Password 附件），从此换机再也不慌。
4 个要点
  • 导出的是「证书 + 私钥」打包：.p12 格式自带导出密码保护，传输丢了也不怕被直接使用
  • 哪些证书要导：Apple Distribution（若是本地版）和 Developer ID Application；开发证书不用导（可重建）
  • 导出置灰怎么办：先解锁钥匙串；仍置灰 = 本机没有私钥，这台机器导不出，只能走路线 C
  • 完整步骤：见第 05 节实操两页（旧机导出 → 新机导入）
03

路线 C · 直接重建

三条路线 · 3 / 3 · 兜底方案
🔄
路线 C · 旧机已不在
revoke 旧证书，
新机重新生成
旧电脑已经抹掉 / 不在手上、私钥拿不回来了？没关系——大多数证书可以撤销重建，新机登录账户让 Xcode 重新生成即可。
心理负担清零
不影响已上架 App只影响后续打包
为什么不影响线上：App Store 上架的版本由苹果重新签名，跟你本地证书无关；revoke 只是让旧私钥作废。
4 个要点
  • 操作入口：新机 Xcode 登录账户 → 项目签名页勾 Automatically manage signing → 缺私钥时按提示 revoke 重建
  • 开发证书：随便重建，苹果官方就推荐每台机器各建一个
  • 发布证书：可以重建，线上 App 无感；正在跑的 CI 要同步换新证书
  • ⚠️ 例外：Developer ID 不能走这条路——revoke 会影响已分发到用户手里的 Mac App（详见下一节）
04

Apple Development

证书类型 · 1 / 3 · 日常开发调试
🛠️
待遇：随便丢
不用备份，
不用迁移
真机调试、日常开发用的证书。苹果官方态度明确：每台机器各建一个就行，开自动签名让 Xcode 自己生成。
换机动作
无 · 登录账户即可
一句话：这是「耗材」级证书——旧机那个不用管，过期自动作废。
3 个要点
  • 数量上限宽松：允许同团队同成员持有多个开发证书，每台 Mac 一个是设计内的正常状态
  • 新机自动生成：Xcode 登录账户、打开项目勾自动签名，第一次构建时自动创建
  • 旧机的不用清理：留着无害，过期自动失效；强迫症可以到开发者后台手动 revoke
04

Apple Distribution

证书类型 · 2 / 3 · 上架 App Store
🏪
待遇：优先云签名
大概率
什么都不用做
上架 App Store 用的发布证书。Xcode 13 起默认走云签名——私钥存苹果云端，Organizer 分发时自动取用，换机零迁移。
换机动作
云签名 · 无本地版 · 导 .p12 或重建
怎么判断自己是哪种：开发者后台 Certificates 列表里标着 「Cloud Managed」 的就是云签名证书，本机无私钥。
3 个要点
  • 云签名（cloud-managed）：到期前 90 天自动轮换新证书，连续签新包无感；这是苹果现在主推的形态
  • 本地版发布证书：丢了可以 revoke 重建，不影响线上 App，只影响后续打包上传
  • 团队协作注意：云签名默认只有 Admin / Account Holder 能用；要让 Developer 角色上传，需在 App Store Connect 勾「Access to Cloud Managed Distribution Certificate」
04

Developer ID

证书类型 · 3 / 3 · App Store 之外分发 Mac App
💎
待遇：传家宝
必须备份，
不可随意 revoke
在 App Store 之外直接发 .dmg / .zip 给用户（如独立分发 Mac 工具），靠它签名 + 公证。这是唯一一类丢了真会肉疼的证书。
换机动作
导 .p12 · 必做异地备份 · 必做
苹果官方原话级纪律：每团队只建一个、私钥当传家宝保管、永远不要轻易 revoke。
3 个要点
  • 每团队上限 5 个，且创建后不能像其他证书那样随手撤销重来——额度极其珍贵
  • revoke 的代价：会影响已分发到用户手里的 App 的信任链，只有私钥泄露时才该走这一步
  • 换机前必做：导出 .p12 + 存一份到密码管理器；这一项做完，换机才算「安全」

实操 · 旧电脑导出 .p12

路线 B 动作分解 · 1 / 2

工具只有一个：「钥匙串访问」（Keychain Access），Xcode 16 起没有别的导出入口。

STEP 1
打开「钥匙串访问」→ 选「登录」钥匙串 →「我的证书」分类
→
STEP 2
找到 Apple Distribution / Developer ID Application，点开三角形确认证书下挂着私钥
→
STEP 3
右键 →「导出」→ 格式选 .p12（个人信息交换）
→
STEP 4
设一个导出密码（导入时要用，记进密码管理器）
→
STEP 5
隔空投送 / U 盘传到新机，同时存一份永久备份
⚠️ 「导出」菜单置灰？① 先点左上角解锁钥匙串；② 仍置灰 = 条目下没有私钥（只有证书壳），这台机器导不出 .p12——只能走路线 C 重建。
05

实操 · 新电脑导入验证

路线 B 动作分解 · 2 / 2

导入只要双击，剩下的交给 Xcode 自动签名。

STEP 1
双击 .p12，输入导出密码 → 证书 + 私钥进入登录钥匙串
→
STEP 2
Xcode → Settings → Accounts → 登录 Apple 账户
→
STEP 3
项目签名页勾 Automatically manage signing，描述文件自动重新下载
→
STEP 4
打一次 Archive 验证全链路：签名不报错 = 迁移完成
💡 描述文件（provisioning profiles）全部在苹果服务器，永远不需要手动搬——Xcode 登录后自动拉取。
06

证书之外：换机清单

四件容易忘的小事
清单 01 · 🪪
公证凭证 notarytool
之前跑过 notarytool store-credentials 的，凭证在旧机钥匙串。新机用 App 专用密码（appleid.apple.com 生成）重新执行一次即可。
清单 02 · 📱
双重认证受信设备
抹掉旧 Mac 之前，确认手机或新 Mac 已是受信任设备、能收验证码。别把唯一能过双重认证的设备抹了。
清单 03 · 🧹
旧电脑交出去之前
确认新机能正常 Archive 后再处理旧机：抹掉所有内容与设置（私钥随钥匙串销毁）；若旧机给别人用不抹机，至少手动删掉发布类私钥并退出 Xcode 账户。
清单 04 · 💳
这些跟换机完全无关
年费续订、App 审核状态、TestFlight、销售报表——都在 App Store Connect 网页端，任何浏览器登录都能管，不依赖任何一台 Mac。

三句话速记

换机前只要把这三句想明白，就不会出事。

账号
账号在云端，跟 Apple 账户走——新机登录即恢复，App Store Connect 一切照旧。
私钥
唯一的本地资产是钥匙串里的私钥——迁移助理整机搬，或钥匙串导 .p12，5 分钟搞定。
例外
Developer ID 是传家宝——上限 5 个、revoke 伤用户，换机前必须导 .p12 异地备份。
